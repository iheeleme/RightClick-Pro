# GitHub Release API Research

## Topic

Use GitHub as the source of truth for RightClick Pro update checks.

## Sources

* GitHub REST API docs: https://docs.github.com/en/rest/releases/releases#get-the-latest-release
* GitHub latest release link docs: https://docs.github.com/en/repositories/releasing-projects-on-github/linking-to-releases

## Findings

* The official REST endpoint is `GET /repos/{owner}/{repo}/releases/latest`.
* GitHub defines this as the latest published full release: non-draft and non-prerelease.
* Public repository release information can be requested without authentication.
* The recommended response fields for this feature are:
  * `tag_name`: version tag, usually like `v1.2.3`.
  * `html_url`: browser URL for the release page.
  * `published_at`: release publication timestamp.
  * `assets[].browser_download_url`: direct asset download links when needed later.
* GitHub supports a web shortcut for the latest release page: `https://github.com/<owner>/<repo>/releases/latest`.
* Current probe for `iheeleme/RightClick-Pro` returned HTTP 404 from `/releases/latest`, so the app must treat "no full release yet" as a normal, user-readable state instead of an error crash.

## Repo Constraints

* README already points users to `https://github.com/iheeleme/RightClick-Pro/releases`.
* Packaged app version comes from `CFBundleShortVersionString`, generated by `scripts/package-macos.sh`.
* Debug/local builds can report `0.0.0-dev`; update comparison should avoid treating dev suffixes as newer stable releases.

## Recommended MVP

Add a manual update-check panel in the Overview settings screen:

* Fetch `https://api.github.com/repos/iheeleme/RightClick-Pro/releases/latest`.
* Compare the latest release `tag_name` with `Bundle.main` short version.
* Show one of four states: unchecked, checking, up-to-date, update available, unavailable/no release.
* Offer an "Open Releases" or "Download" button that opens GitHub in the browser.

## Edge Cases

* HTTP 404: no full release available yet.
* Network offline or timeout: show retryable warning.
* Malformed version/tag: show latest tag but do not claim an update is available.
* Rate limit/403: show GitHub request failed message without asking for tokens.
